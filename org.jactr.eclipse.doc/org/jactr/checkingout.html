Checking out jACT-R

Maintaining on the cusp of development is always fun, but it can be a little complex. The first thing you need is to be familiar with CVS. All code is hosted by SourceForge.

Anonymous Source Access

Anonymous access allows you to checkout, but never commit any changes.

pserver:anonymous@cvs.sourceforge.net:/cvsroot/jactr

Developer Source Access

Developer access allows you to checkout and commit any changes.

extssh:developername@cvs.sourceforge.net:/cvsroot/jactr

CVS Layout

The CVS layout is a tad odd. The reason for this is to support as many different development environments with as little redundancy as possible. There is a directory for each (currently) support building tool, libraries, source, and tests.

  • src : contains the source for the contained projects of the form src/project.name/src/programming-language. (i.e. src/org.jactr/src/java)
  • test : same as above but only contains testing code
  • repository : all the libraries and jars required by specific projects of the form repository/project.name/lib/. (i.e. repository/org.jactr/lib/...)
  • eclipse : Eclipse project descriptions for the supported projects.
  • ant : ant build files for command line building
  • maven : maven configuration files.

In order to actually use this, first checkout the files for your given build tool (Eclipse, ant, maven). Then, in the source directory for that configuration, check out the source files for the appropriate project. Do the same for the repository content.

Example

Let's use Eclipse to check out the core jACT-R code.

  1. Open the CVS Repositories View (Window-Show View-Other-CVS-CVS Repository)
  2. Add a new repository (Right click-New-Repository Location)
  3. Enter the connection information from above
  4. Select Head-eclipse-org.jactr. Check Out As..
  5. Check out as project, name: org.jactr
  6. Select Head-src-org.jactr-src-java. Check Out As..
  7. Check out into existing project, target folder name: org.jactr
  8. Select Head-repository-org.jactr-lib. Check Out As..
  9. Check out into existing project, target folder name: org.jactr
  10. You should now be able to build without any problems
